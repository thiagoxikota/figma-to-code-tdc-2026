Prompt 1 · ler o frame
Leia o frame "Tela demo" deste arquivo Figma. Devolva a árvore de camadas, os componentes usados e as variáveis (tokens) aplicadas em cada elemento. Não descreva pixels: descreva estrutura e semântica.
No Claude Desktop, conversa nova, com o Bridge aberto no arquivo.
Ele pede estrutura e tokens, não pixels. A resposta vira auditável.

Prompt 2 · gerar o componente
Gere um componente Card em React + TypeScript, function component. Props tipadas: title: string, description: string, imageUrl?: string, onClick?: () => void. Cubra os estados default, hover, focus-visible e disabled. Acessibilidade: elemento article, heading por prop, foco visível de 2px, alvo mínimo de 44px, contraste mínimo AA. Sem libs externas.
Antes de gerar, leia o frame "Tela demo" deste arquivo Figma via MCP e use os tokens (cor, espaçamento, tipografia, raio) e a estrutura de lá. Não invente valores; puxe do Figma. Devolva só o arquivo Card.tsx.
Mesma spec de sempre, com uma linha a mais: puxe a verdade do Figma. É isso que tira o código da média.

Validação e correção
Corrija uma falha. Volte ao frame, pegue o token certo, confira que bateu.
Confira a conexão e o token:
npm run workshop:checknpm run workshop:doctor
Se quiser recomeçar do zero, npm run workshop:reset restaura os componentes sem tocar na sua credencial.
Pronto aqui é o ponto corrigido batendo com o frame. Não o card inteiro pronto para produção.

Limpeza e kit
Antes de sair da sala.
  1. Revogue o token no Figma, em Settings, Security.
  2. Remova o MCP: npm run workshop:setup-mcp -- --remove
  3. Remova o plugin Bridge, se quiser.
  4. Na máquina do evento, limpe o projeto e saia das contas.
